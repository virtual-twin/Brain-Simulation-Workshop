1 Dynamical Systems

This notebook is the static companion to the workshop slides. The slides show the same concepts visually; here we make them explicit in TVBO code.

The workshop starts from the generic state-evolution equation

\[ dS_i = \left[f_d(S_i, \theta^d, C_i, I_i)\right]dt + g(S_i, \theta^g)\,dW_i. \]

This notebook first removes everything except the deterministic local dynamics: one node, no coupling, no external input, and no noise. That leaves the core object that appears everywhere in TVBO: a dynamical system with state variables, parameters, initial conditions, and equations of motion.

1.1 Learning Path

  1. Define a system as a TVBO Dynamics object.
  2. Run it through a SimulationExperiment.
  3. Read results with named xarray dimensions.
  4. Compare initial conditions as different trajectories in the same flow.
  5. Use phase space to see the geometry of the dynamics.
  6. Vary parameters and inspect how the behavior changes.
  7. Add damping and load terms to move from a toy model toward realism.
  8. Connect this toy example to neural mass models, networks, events, observations, and bifurcations.

The didactic arc follows the standard dynamical-systems progression: state space and flows, fixed points and stability, parameter-dependent behavior, and then richer systems. That is the same conceptual route used in Strogatz-style nonlinear dynamics and in Izhikevich-style phase-plane neuroscience, but the examples below use TVBO objects throughout.

1.2 From a Second-Order Equation to TVBO

A mass \(m\) attached to a spring with stiffness \(k\) satisfies

\[ m\ddot{x} = -kx. \]

Numerical ODE solvers work with first-order systems, so we introduce velocity \(v = \dot{x}\):

\[ \dot{x} = v, \qquad \dot{v} = -\frac{k}{m}x. \]

In TVBO, these become state variables and parameters. State variables evolve over time. Parameters are fixed during one simulation unless we explicitly explore them.

def make_spring(x0=2.0, v0=0.0, k=0.0001, m=1.0):
    return Dynamics(
        name="SpringMass",
        description="Undamped spring-mass oscillator in first-order form.",
        state_variables={
            "x": {
                "description": "Displacement from equilibrium",
                "unit": "m",
                "equation": {"rhs": "v"},
                "initial_value": x0,
            },
            "v": {
                "description": "Velocity",
                "unit": "m/ms",
                "equation": {"rhs": "-(k/m) * x"},
                "initial_value": v0,
            },
        },
        parameters={
            "k": {
                "description": "Spring stiffness",
                "unit": "kg/ms^2",
                "value": k,
            },
            "m": {
                "description": "Mass",
                "unit": "kg",
                "value": m,
            },
        },
    )

1.3 Run a Simulation Experiment

TVBO separates the model from the experiment. A Dynamics object describes the equations; a SimulationExperiment says how to run them.

def run_dynamics(dynamics, duration=900.0, step_size=1.0):
    experiment = SimulationExperiment(dynamics=dynamics)
    experiment.integration.duration = duration
    experiment.integration.step_size = step_size
    return experiment.run()


def run_spring(x0=2.0, v0=0.0, k=0.0001, m=1.0, duration=900.0, step_size=1.0):
    return run_dynamics(make_spring(x0=x0, v0=v0, k=k, m=m), duration=duration, step_size=step_size)


def series(result, variable):
    return result.integration.data.sel(variable=variable).squeeze(drop=True)


def plot_series(result, variable="x", ax=None, label=None, **kwargs):
    if ax is None:
        _, ax = plt.subplots(figsize=(6, 3.2))
    data = series(result, variable)
    ax.plot(data.coords["time"].to_numpy(), data.to_numpy(), label=label or variable, **kwargs)
    return ax

1.4 Initial Conditions: Same Flow, Different Trajectories

Initial conditions pick the starting point in state space. They do not change the vector field. This distinction matters later for neural models: changing a state variable’s initial value is not the same as changing a physiological parameter.

For the undamped spring, larger initial displacement gives a larger-amplitude orbit, but the frequency remains

\[ \omega = \sqrt{\frac{k}{m}}. \]

1.5 Phase Space: The Geometry of the Flow

A time series shows one coordinate as a function of time. A phase portrait shows the full state \((x, v)\). This is the geometry-first view emphasized in nonlinear dynamics and in phase-plane neuroscience.

For the undamped spring, the trajectories are closed loops because energy is conserved. The vector field is the same for every trajectory; only the initial condition changes.

fig, ax = plt.subplots(figsize=(5.2, 4.8))
make_spring().plot(kind="vectorfield", ax=ax, alpha=0.25)

for x0, color in zip([0.5, 1.0, 1.5, 2.0], ["C0", "C1", "C2", "C3"]):
    phase_result = run_spring(x0=x0, duration=900.0, step_size=1.0)
    x = series(phase_result, "x")
    v = series(phase_result, "v")
    ax.plot(x.to_numpy(), v.to_numpy(), color=color, lw=1.4, label=rf"$x_0={x0}$")
    ax.scatter(x.isel(time=0).to_numpy(), v.isel(time=0).to_numpy(), color=color, s=24, zorder=3)

ax.set(xlabel="Position $x$", ylabel="Velocity $v$", title="Phase space")

1.6 Parameters: Changing the System Itself

Parameters are different from initial conditions. Initial conditions choose where a trajectory begins; parameters reshape the vector field.

Here we vary the mass \(m\) while keeping \(k\) and \(x_0\) fixed. Heavier mass means more inertia, so the oscillation is slower.

fig, ax = plt.subplots(figsize=(6, 3.2))
k_fixed = 0.0001

for mass, color in zip([0.5, 1.0, 2.0], ["C0", "C1", "C2"]):
    mass_result = run_spring(x0=-2.0, k=k_fixed, m=mass, duration=1200.0, step_size=1.0)
    omega = np.sqrt(k_fixed / mass)
    plot_series(mass_result, "x", ax=ax, label=rf"$m={mass}$, $\omega={omega:.4f}$", color=color)

For larger parameter sweeps, the TVBO docs generalize this pattern with Exploration and ExplorationAxis, so the sweep becomes part of the experiment specification rather than a manual Python loop.

from tvbo.datamodel.schema import Exploration, ExplorationAxis

experiment = SimulationExperiment(dynamics=make_spring(x0=-2.0))
experiment.explorations["mass_sweep"] = Exploration(
    name="mass_sweep",
    space=ExplorationAxis(parameter="m", explored_values=[0.5, 1.0, 2.0]),
)
mass_result = experiment.run("tvboptim")
mass_result.explorations["mass_sweep"].plot(overlay=True)

1.7 Toward Realism: Damping and Load

The ideal spring never loses energy. Real systems usually do. We can extend the same TVBO model by adding a damping term and a constant load:

\[ \dot{x} = v, \qquad \dot{v} = -\frac{k}{m}x - \frac{c}{m}v + g_{\mathrm{eff}}. \]

The damping term changes the transient. The load shifts the equilibrium from \(x = 0\) to

\[ x^* = \frac{m g_{\mathrm{eff}}}{k}. \]

def make_damped_loaded_spring(x0=2.0, v0=0.0, k=0.0001, m=1.0, c=0.006, g_eff=0.00015):
    system = make_spring(x0=x0, v0=v0, k=k, m=m)
    system.description = "Spring-mass oscillator with damping and a constant load."
    system.state_variables["v"]["equation"]["rhs"] = "-(k/m) * x - (c/m) * v + g_eff"
    system.parameters["c"] = Parameter(name="c", description="Damping coefficient", value=c)
    system.parameters["g_eff"] = Parameter(name="g_eff", description="Constant effective load", value=g_eff)
    return system


ideal = make_spring(x0=2.0)
realistic = make_damped_loaded_spring(x0=2.0)

ideal_result = run_dynamics(ideal, duration=1800.0, step_size=1.0)
realistic_result = run_dynamics(realistic, duration=1800.0, step_size=1.0)

x_eq_real = realistic.parameters["m"].value * realistic.parameters["g_eff"].value / realistic.parameters["k"].value

fig, (ax_time, ax_phase) = plt.subplots(1, 2, figsize=(10, 3.6))

plot_series(ideal_result, "x", ax=ax_time, label="ideal", color="C0")
plot_series(realistic_result, "x", ax=ax_time, label="damped + load", color="C3")
ax_time.axhline(0.0, color="C0", lw=0.8, ls="--", alpha=0.6)
ax_time.axhline(x_eq_real, color="C3", lw=0.8, ls="--", alpha=0.6)
ax_time.set(xlabel="Time", ylabel="Displacement $x$", title="Time domain")
ax_time.legend(handlelength=1)

for label, sim_result, color in [("ideal", ideal_result, "C0"), ("damped + load", realistic_result, "C3")]:
    x = series(sim_result, "x")
    v = series(sim_result, "v")
    ax_phase.plot(x.to_numpy(), v.to_numpy(), color=color, lw=1.4, label=label)

ax_phase.scatter([0.0, x_eq_real], [0.0, 0.0], color=["C0", "C3"], s=28, zorder=3)
ax_phase.set(xlabel="Position $x$", ylabel="Velocity $v$", title="Phase-space geometry")

1.8 Connection to TVBO Documentation

The spring example is intentionally simple, but each piece maps directly to the main TVBO documentation topics.

Concept in this notebook TVBO object or guide Why it matters later
Equations and parameters Dynamics, DefineDynamicalSystem Neural mass and oscillator models use the same schema.
Run configuration SimulationExperiment The experiment combines dynamics, integration, networks, observations, algorithms, and continuations.
Named output ExperimentResult.integration.data Results are xarray objects, so analysis should use named dimensions like time, variable, and node.
Initial conditions InitialConditions Defaults, node overrides, sampled distributions, and transient warm-up resolve the initial state.
Parameter sweeps Exploration Parameter scans become declarative experiment metadata.
Phase portraits BifurcationTutorial Geometry explains stability, oscillations, and bifurcations.
Damping and load Model specification More realistic assumptions are added by changing equations and parameters, not by changing the execution workflow.

1.9 From Spring-Mass to Neural Mass Models

The next workshop section moves from a physical toy system to neural mass models. The TVBO database already stores many such models with the same structure: state variables, parameters, equations, and metadata.

neural_mass_models = Dynamics.list_db(model_type="neural_mass")

This is the key bridge: the spring has \((x, v)\), while a neural mass model has variables such as membrane potentials, firing-rate transforms, or synaptic gating variables. But the TVBO workflow remains the same:

  1. Choose or define Dynamics.
  2. Put it in a SimulationExperiment.
  3. Run the experiment.
  4. Analyze result.integration.data and any observations.

1.10 What Comes Next in the Workshop

The spring notebook stops at a single deterministic system. The later slides add the missing terms from the workshop equation.

Next concept TVBO object Role
Brain regions and structural connectivity Network Adds nodes, edges, weights, delays, and positions.
Inter-regional influence Coupling Turns source-node activity into input for target-node dynamics.
External stimulation or perturbations Event Adds time-dependent input \(I_i(t)\).
Measurement models observations Transforms hidden state into BOLD, FC, PSD, or custom outputs.
Noise noise terms and stochastic integrators Adds fluctuations via \(g(S_i, \theta^g)dW_i\).
Regime changes Continuation Tracks fixed points, stability changes, and oscillatory branches.

This is why phase space and stability are introduced early. They are not only mathematical background; they are the language used later to interpret excitability, oscillations, bifurcations, and transitions in neural population models.
